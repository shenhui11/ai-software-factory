# Coding Rules

## 命名规范
- 变量名必须语义清晰
- 不允许使用 tmp、data、obj 等无意义命名
- 类名使用 PascalCase
- 函数名使用 snake_case

## 函数设计
- 单个函数不超过 80 行
- 一个函数只做一件事
- 必须有明确输入和返回

## 类型规范
- 必须使用类型注解（Python）
- 所有公共函数必须标注返回类型

## 错误处理
- 不允许裸 except
- 必须返回明确错误信息
- API 必须有统一错误结构

## 结构规范
- 不允许把所有逻辑写在一个文件
- 按功能拆分模块

## 禁止事项
- 不允许硬编码 token / 密码
- 不允许 print 调试（使用 logging）
- 不允许跳过测试

## 测试约束
- 每个核心函数必须有测试
- 不允许只写 happy path
