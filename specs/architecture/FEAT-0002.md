# Architecture - FEAT-0002

## 设计目标
- 解耦“套餐规则”和“账户剩余额度”。
- 为“日免费章节、月免费章节、赠送免费章节”建立清晰的后端模型。
- 让后台页面围绕“默认套餐 + 目标账号额度账户”展开。
- 保持对现有会员、订单、调额逻辑的最小侵入式改造。

## 数据模型调整

### MembershipPlan
现有：
- `id`
- `name`
- `free_chapter_quota`
- `monthly_quota`
- `description`

调整后：
- `id`
- `name`
- `daily_free_chapters`
- `monthly_free_chapters`
- `description`

说明：
- `free_chapter_quota` 废弃，统一迁移为 `daily_free_chapters`
- `monthly_quota` 重命名为 `monthly_free_chapters`

### UserQuota
现有：
- `free_remaining`
- `monthly_remaining`

调整后：
- `daily_remaining`
- `monthly_remaining`
- `bonus_remaining`
- `last_daily_reset_at`
- `last_monthly_reset_at`

说明：
- `daily_remaining` 表示用户今天还能使用多少章节。
- `monthly_remaining` 表示用户本月还能使用多少章节。
- `bonus_remaining` 表示后台赠送的附加章节池。
- `last_daily_reset_at` 与 `last_monthly_reset_at` 用于惰性重置。

## 核心规则

### 1. 初始化规则
新用户首次访问额度时：
- 绑定默认套餐 ID
- `daily_remaining = plan.daily_free_chapters`
- `monthly_remaining = plan.monthly_free_chapters`
- `bonus_remaining = 0`
- `last_daily_reset_at = now`
- `last_monthly_reset_at = now`

### 2. 惰性重置规则
在每次读取或消费用户额度前执行：
- 若当前日期与 `last_daily_reset_at` 不同，则将 `daily_remaining` 重置为当前套餐的 `daily_free_chapters`
- 若当前月份与 `last_monthly_reset_at` 不同，则将 `monthly_remaining` 重置为当前套餐的 `monthly_free_chapters`

说明：
- 推荐按账号当前时区或系统统一时区执行；首期可统一用系统时区。

### 3. 扣减规则
消费 `requested` 章节时：
1. 先扣 `daily_remaining`
2. 再扣 `monthly_remaining`
3. 最后扣 `bonus_remaining`

若三者总和不足，则抛出 `QUOTA_EXCEEDED`。

### 4. 默认套餐切换规则
- 系统保留一个 `default_plan_id`
- 切换默认套餐仅影响：
  - 新用户初始化
  - 全局默认账户视角展示
  - 未做显式自定义套餐重置的未来刷新逻辑（如产品后续决定启用）

首期建议：
- 不自动覆盖已有用户的当前剩余额度
- 不自动回填老用户的 `daily_remaining` / `monthly_remaining`
- 仅在下次重置时按用户绑定套餐规则恢复

## API 调整

### `GET /api/membership/quotas`
返回结构调整为：

```json
{
  "quota": {
    "daily_remaining": 3,
    "monthly_remaining": 12,
    "bonus_remaining": 5
  },
  "default_plan": {
    "id": "plan-basic",
    "name": "基础套餐",
    "daily_free_chapters": 5,
    "monthly_free_chapters": 50,
    "description": "适合日常创作"
  },
  "user_plan": {
    "id": "plan-basic",
    "name": "基础套餐",
    "daily_free_chapters": 5,
    "monthly_free_chapters": 50,
    "description": "适合日常创作"
  }
}
```

说明：
- 若当前页面只需默认套餐概览，可直接读 `default_plan`
- 若后续需要支持用户绑定自定义套餐，可保留 `user_plan`

### `GET /admin/memberships`
返回结构建议为：

```json
{
  "plans": [],
  "default_plan_id": "plan-basic",
  "default_plan": {},
  "quota": {
    "daily_remaining": 0,
    "monthly_remaining": 0,
    "bonus_remaining": 0
  },
  "target_user_id": "user_xxx"
}
```

### `POST /admin/memberships/plans`
请求体：
- `name`
- `daily_free_chapters`
- `monthly_free_chapters`
- `description`

### `PATCH /admin/memberships/plans/{plan_id}`
同上。

### `POST /admin/memberships/plans/{plan_id}/activate`
语义调整为：
- 设置默认套餐
- 路由可保留，但前端文案改为“设为默认套餐”

### `POST /admin/quotas/adjust`
请求体建议调整为：
- `target_user_id`
- `daily_delta`
- `monthly_delta`
- `bonus_delta`

说明：
- `bonus_delta` 用于发放或扣减赠送免费章节
- `daily_delta` 与 `monthly_delta` 代表人工修正剩余额度

## 前端展示设计

### 额度概览卡
字段：
- 默认套餐：显示 `default_plan.name`
- 日免费章节：显示 `default_plan.daily_free_chapters`
- 月免费章节：显示 `default_plan.monthly_free_chapters`
- 赠送免费章节：显示 `quota.bonus_remaining`
- 当日剩余：显示 `quota.daily_remaining`
- 当月剩余：显示 `quota.monthly_remaining`
- 套餐总数：显示 `plans.length`

### 套餐列表
每张卡片显示：
- 套餐名称
- 套餐 ID
- 日免费章节
- 月免费章节
- 描述
- 当前默认徽标
- `编辑套餐`
- `设为默认套餐`

### 套餐编辑弹框
组件建议：
- `PlanEditorModal`

状态：
- `isOpen`
- `mode: create | edit`
- `editingPlanId`
- `form`

交互：
- 从列表点击“编辑套餐”时打开弹框
- 从“新建套餐”按钮点击时打开空表单
- 保存成功后关闭弹框并刷新后台数据

## 数据迁移建议
- `MembershipPlan.free_chapter_quota -> daily_free_chapters`
- `MembershipPlan.monthly_quota -> monthly_free_chapters`
- `UserQuota.free_remaining -> bonus_remaining`
- `UserQuota.monthly_remaining -> monthly_remaining`
- 新增 `daily_remaining` 初始值：
  - 若有用户绑定套餐，取该套餐 `daily_free_chapters`
  - 否则取默认套餐 `daily_free_chapters`
- 新增 `last_daily_reset_at` / `last_monthly_reset_at` 初始值为迁移执行时间

说明：
- 这是偏保守迁移方案，优先保留历史“额外剩余额度”价值。

## 测试重点
- 默认套餐概览与套餐列表一致性
- 套餐编辑弹框的打开、保存、刷新
- 三段式额度扣减顺序
- 日/月惰性重置
- 赠送免费章节在日/月耗尽后的兜底消费
