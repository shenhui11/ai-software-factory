## 最终策略
## Agent Roles

### 1. 故事编辑 Agent
- 输入：章节目标、上文摘要、人物设定、世界观约束
- 输出：3 个剧情走向方案
- 要求：每个方案包含本章核心冲突、关键事件、结尾钩子

### 2. 资深主编 Agent
- 输入：3 个剧情走向方案
- 输出：每个方案评分 + 说明 + 最终选中方案
- 评分维度：
  - 情节推动
  - 故事一致性
  - 吸引力
- 决策：选择最高分方案进入正文生成

### 3. 写手 Agent
- 输入：选中的剧情走向、上下文设定
- 输出：章节正文

### 4. 资深读者 Agent
- 输入：章节正文
- 输出：评分 + 问题说明 + 是否回流
- 最多回流 5 次
- 最终保留评分最高版本

## End-to-End Flow

1. 用户输入续写任务
2. 系统一次最多生成 10 章
3. 每一章执行以下流程：
   1) 故事编辑 Agent 生成 3 个走向方案
   2) 资深主编 Agent 对 3 个方案评分并选择最高分方案
   3) 写手 Agent 基于选中方案生成正文
   4) 资深读者 Agent 对正文评分并给出问题
   5) 若评分未达标，则回流给写手 Agent 重写
   6) 最多回流 5 次
   7) 保留本章评分最高的正文作为最终版本
4. 所有章节完成后输出结果

## Scoring Rules

### 主编评分
- 满分：10 分
- 维度：
  - 情节推动：0-10
  - 故事一致性：0-10
  - 吸引力：0-10
- 最终分数：平均分

### 读者评分
- 满分：10 分
- 维度：
  - 可读性：0-10
  - 情绪张力：0-10
  - 连贯性：0-10
- 最终分数：平均分

### 通过阈值
- 章节正文评分 >= 8.0 则直接通过
- < 8.0 则进入回流
- 最多回流 5 次
- 若 5 次后仍未达标，选最高分版本输出，并标记为人工复核建议

## MVP Scope
首期必须实现：
- 单本小说续写
- 单次最多续写 10 章
- 每章 3 个走向候选
- 主编评分选择
- 写手正文生成
- 读者评分回流
- 最多回流 5 次
- 保留最高分版本
- 基础任务状态展示

首期不做：
- 多用户协作
- 人工在线逐章修改台
- 高级提示词运营后台
- 细粒度成本统计
- 复杂队列调度

## Technical Decision

- 后端：Node.js / Python
- 任务编排：BullMQ / Celery / 本地串行 Job Runner
- 数据库存储：PostgreSQL / SQLite
- LLM 调用：统一 AgentRunner 封装
- 每章处理采用串行，避免上下文混乱
- 单次任务最大章节数限制为 10
- 每章内部 revision 最大 5 次
