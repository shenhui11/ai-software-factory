# PRD - FEAT-0001

## 背景
“智能小说协同生成平台”面向小说创作者，目标是提供一个由多类 AI Agent 协同执行、但仍保留用户最终决策权的创作环境。产品核心问题不是单次文本生成，而是围绕中长篇小说创作中的情节连贯性、章节可控性、质量回流和模板化运营，建立可持续的生产机制。

基于当前需求澄清，首期产品形态为 Web 平台，覆盖创作前台与运营后台。前台支持用户创建小说项目、选择题材/篇幅/模板、编辑总纲与章节大纲、触发章节生成与改写；后台支持模板审核、会员套餐配置、订单管理、手动调额、内容安全策略与日志留存等能力。

## 最终选定方案
最终采用 `solution_1 - 审稿驱动型章节工坊` 作为首发方案。

该方案以章节为中心构建创作工作台，核心流程是“故事编辑 Agent 出候选方案 -> 用户或系统选定方案 -> 写手 Agent 生成正文 -> 资深读者 Agent 评分 -> 低分回流 -> 保留最佳结果”。该方案与当前已明确的逐章审核、质量闭环、人工拍板机制最一致，MVP 范围清晰，适合快速落地。

尽管 `scoring` 中推荐的是 `solution_2`，但本 PRD 严格以人工决策文件 `specs/decision/FEAT-0001.md` 为准。

## 方案修改点
在 `solution_1` 基础上，人工决策要求补充和固定以下改动：

1. 除默认人工确认流程外，首期必须支持全自动流程。
2. 每章开头由故事编辑 Agent 生成 3 个剧情走向方案。
3. 新增“资深主编 Agent”，对 3 个剧情走向按“情节推动、故事一致性、吸引力”评分，并自动选择最高分方案进入正文生成。
4. 写手 Agent 基于入选方案生成章节正文。
5. 资深读者 Agent 对章节正文进行评分与问题说明，若未达阈值则最多回流 5 次。
6. 每章最终保留评分最高的正文版本；若 5 次后仍未达标，输出最高分版本并标记“建议人工复核”。
7. 单次任务最高支持连续续写 10 章，章节间采用串行执行，避免上下文混乱。
8. 连贯性记忆采用结构化记忆方式，至少包含全书摘要、角色卡、世界观设定、已发生事件摘要和章节状态。
9. 段落级“重写/扩写”必须包含上下文一致性检查与差异对比。

## 目标用户
- 个人网文作者、轻小说作者，重视章节质量与创作可控性。
- 需要快速产出初稿但仍希望保留人工审核权的内容工作者。
- 平台运营人员，需要管理模板、会员套餐、免费额度、订单与内容安全策略。
- 偏好自动续写但又需要质量闸门与异常复核提示的创作者。

## 核心业务目标
- 提供可持续的章节式小说创作工作流，而不是一次性文本生成工具。
- 通过多 Agent 分工与质量回流机制，提高章节质量与情节连贯性。
- 同时支持人工确认模式与全自动模式，兼顾可控创作与高效率续写。
- 通过模板与会员体系支撑平台运营与商业化。
- 为未来扩展配图、社区、数据看板、多格式导出等能力保留接口空间，但不进入首期交付。

## MVP 功能范围
- 小说项目创建：支持选择题材、篇幅、模板，并初始化项目基础设定。
- 模板体系：支持系统模板与用户自定义模板的创建、编辑、使用；后台支持模板审核与发布。
- 大纲编辑：支持总纲与章节大纲编辑，用户可查看与调整章节目标。
- 故事编辑协作：针对单章生成 3 个剧情走向候选方案，每个方案包含核心冲突、关键事件、结尾钩子。
- 主编评分选案：资深主编 Agent 对 3 个候选方案评分并自动选择最高分方案。
- 正文生成：写手 Agent 基于已选方案与上下文生成章节正文。
- 质量评估回流：资深读者 Agent 按可读性、情绪张力、连贯性评分；低于阈值则回流重写，单章最多 5 次。
- 最佳版本保留：每章保留评分最高的正文版本，未达阈值时也需输出最佳版本并标记人工复核建议。
- 连续续写：单次任务最多支持连续生成 10 章，章节按顺序执行。
- 创作模式：支持人工确认模式和全自动模式。
- 段落级重写/扩写：支持对已生成正文局部发起重写或扩写，并展示差异对比与一致性检查结果。
- 结构化记忆：支持全书摘要、角色卡、世界观卡、已发生事件和章节状态等长期记忆。
- 会员与后台运营：支持免费章节额度、套餐配置、月度额度调整、订单管理、手动调额、基础运营报表、敏感内容策略配置、日志留存。
- 内容安全与合规：支持敏感题材过滤、版权声明与数据留存要求。

## 非目标范围
- 多用户实时协作写作。
- 社区发布、评论、点赞、共创协作等社交能力。
- AI 配图生成与图文联动。
- 多格式导出（PDF、EPUB、DOCX 等）。
- 复杂支付结算、发票、渠道分销和营销活动系统。
- 高级 BI 分析和复杂成本统计。
- 企业级权限体系与复杂审批流。
- 完整移动端 App 与离线创作体验。

## 关键用户流程
### 流程一：人工确认模式
1. 用户创建小说项目，选择题材、篇幅和模板。
2. 用户完善总纲、角色设定、世界观与章节目标。
3. 用户对某一章节发起生成。
4. 故事编辑 Agent 输出 3 个剧情走向方案。
5. 资深主编 Agent 对 3 个方案评分并推荐最高分方案。
6. 用户查看候选方案与评分说明，确认采用某一方案。
7. 写手 Agent 生成章节正文。
8. 资深读者 Agent 对正文评分并给出问题说明。
9. 若分数低于阈值，则系统回流重写，单章最多 5 次。
10. 用户确认最终版本入库，后续可对任意段落执行重写或扩写。

### 流程二：全自动连续续写模式
1. 用户输入续写任务并开启全自动模式。
2. 用户设置目标章节数，单次最多 10 章。
3. 系统按章节顺序串行执行：
4. 故事编辑 Agent 为当前章生成 3 个剧情走向方案。
5. 资深主编 Agent 对 3 个方案评分并自动选择最高分方案。
6. 写手 Agent 生成章节正文。
7. 资深读者 Agent 评分，若未达阈值则回流重写，最多 5 次。
8. 系统保留该章最高分版本，更新项目记忆，再进入下一章。
9. 全部章节完成后输出结果；低于阈值但已达回流上限的章节需标记“建议人工复核”。

## 页面/交互要点
- 项目创建页：支持题材、篇幅、模板选择，并清晰展示模板来源与会员权益消耗。
- 项目工作台：以章节为中心展示章节列表、生成状态、评分结果、是否需要人工复核。
- 总纲/章节编辑器：支持查看和编辑总纲、章节目标、角色设定、世界观约束。
- 候选方案面板：并列展示 3 个剧情走向、主编评分及评语；人工模式下支持用户选择，自动模式下展示系统已选结果。
- 章节审稿台：展示正文、读者评分、问题说明、回流次数、最终保留版本。
- 差异对比视图：段落级重写/扩写后展示修改前后差异，并提示上下文一致性检查结果。
- 自动任务面板：允许用户配置一次续写章节数（最多 10 章）、查看当前执行进度、暂停后续批次。
- 记忆卡片面板：展示全书摘要、角色卡、世界观设定、事件链和章节状态，帮助用户理解上下文依据。
- 后台管理页：覆盖模板审核发布、会员套餐配置、免费额度与月度额度调整、订单管理、手动调额、基础报表、内容安全策略、日志留存。

## Acceptance Criteria
- 用户可以创建小说项目，并在创建时选择题材、篇幅和模板。
- 用户可以创建、编辑并使用系统模板和自定义模板；运营后台可以审核和发布模板。
- 用户可以编辑总纲和章节大纲，并保存角色设定、世界观约束和章节目标。
- 用户对任一章节发起生成时，系统必须产出 3 个剧情走向方案，且每个方案包含核心冲突、关键事件、结尾钩子。
- 资深主编 Agent 必须对 3 个剧情走向分别给出评分和说明，并自动选出最高分方案。
- 在人工确认模式下，用户可以查看候选方案及评分说明，并手动确认进入正文生成的方案。
- 写手 Agent 必须基于选中的方案与结构化上下文生成章节正文。
- 资深读者 Agent 必须对正文按可读性、情绪张力、连贯性评分，并输出问题说明。
- 当正文评分低于通过阈值时，系统必须自动回流重写，单章最大回流次数为 5 次。
- 系统必须保留单章所有版本中的最高分版本作为最终版本。
- 若单章回流达到 5 次后仍未达通过阈值，系统仍需输出最高分版本，并标记“建议人工复核”。
- 全自动模式下，系统必须支持单次最多连续生成 10 章，并按章节顺序串行执行。
- 每完成一章后，系统必须更新结构化记忆，再进入下一章生成。
- 用户可以对已生成正文的任意段落发起重写或扩写，并看到差异对比和上下文一致性检查结果。
- 系统必须维护至少 5 类结构化记忆：全书摘要、角色卡、世界观设定、已发生事件摘要、章节状态。
- 后台必须支持会员套餐配置、免费章节额度配置、月度额度调整、订单管理、手动调额、基础运营报表。
- 系统必须支持敏感内容过滤、版权声明展示和必要的数据留存。
