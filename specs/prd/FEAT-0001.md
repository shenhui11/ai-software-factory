# PRD - FEAT-0001

## 背景
“智能小说协同生成平台”旨在提供一个由多种 AI Agent 协作、并保留用户最终决策权的小说创作环境。首期产品需要优先解决小说续写过程中的情节连贯性、章节质量不稳定、人工创作效率低等问题，同时兼顾模板与会员体系的基础运营能力。

现有设计评估中，`scoring` 曾推荐更偏大纲中枢的方案，但本项目最终以人工在 `specs/decision/FEAT-0001.md` 中确认的方案为准，采用更利于快速落地的“审稿驱动型章节工坊”，并在此基础上补充全自动续写流程。

## 最终选定方案
最终选定方案为 `solution_1 - 审稿驱动型章节工坊`。

产品形态为以章节为中心的 Web 创作工作台，围绕“章节候选方案生成 -> 主编筛选 -> 正文生成 -> 读者评分 -> 低分回流”的闭环展开。系统既支持用户逐章把关，也支持一次最多续写 10 章的全自动模式。

核心 Agent 分工如下：
- 故事编辑 Agent：基于章节目标、上文摘要、人物设定、世界观约束，生成 3 个剧情走向方案。
- 资深主编 Agent：对 3 个剧情走向方案按情节推动、故事一致性、吸引力进行评分，选择最高分方案进入正文生成。
- 写手 Agent：基于选中方案和上下文设定生成章节正文。
- 资深读者 Agent：对正文按可读性、情绪张力、连贯性评分，低于阈值时触发回流，单章最多回流 5 次，并最终保留最高分版本。

## 方案修改点
在原始 `solution_1` 的基础上，必须增加以下修改：
- 增加全自动流程，不仅支持逐章人工确认，也支持系统自动连续生成章节。
- 每章必须先生成 3 个剧情走向方案，不能直接进入正文生成。
- 新增“资深主编 Agent”作为剧情方案评分与筛选节点，负责从 3 个走向方案中选出最高分方案。
- 正文生成后必须由“资深读者 Agent”评分并给出问题说明。
- 若章节正文评分未达标，必须回流给写手 Agent 重写，单章最多回流 5 次。
- 单章最终输出必须保留本轮所有版本中的最高分版本。
- 单次续写任务最高支持连续生成 10 章内容。
- 若 5 次回流后仍未达标，仍输出最高分版本，并标记为建议人工复核。

## 目标用户
- 个人网文作者、轻小说作者，希望提高续写效率，同时保留章节级可控性。
- 希望通过 AI 降低初稿成本、提升批量产出的内容工作室。
- 重视人工拍板、不能接受完全黑盒写作的审核型创作者。
- 需要管理模板、套餐、额度、订单和内容安全策略的平台运营人员。

## 核心业务目标
- 提供一个可持续的小说续写工作流，而不是一次性文本生成工具。
- 通过多 Agent 串行协作，提升章节质量与故事连贯性。
- 通过主编评分与读者评分双重机制，降低低质量章节直接输出的概率。
- 支持人工确认模式与全自动模式，满足不同创作效率和可控性需求。
- 以模板、会员额度和后台运营能力支撑平台商业化基础。

## MVP 功能范围
- 小说项目创建，支持选择题材、篇幅、模板。
- 支持系统模板与用户自定义模板的创建、编辑、使用。
- 提供总纲与章节大纲编辑能力，用于支撑后续章节续写。
- 用户可发起单本小说续写任务，单次任务最多续写 10 章。
- 每章由故事编辑 Agent 生成 3 个剧情走向候选方案。
- 资深主编 Agent 对 3 个候选方案评分，并自动选择最高分方案。
- 写手 Agent 基于选中方案生成章节正文。
- 资深读者 Agent 对章节正文评分、输出问题说明，并在低于阈值时触发回流。
- 单章最多回流 5 次，最终保留评分最高的正文版本。
- 提供基础任务状态展示，展示章节处理进度、评分结果、是否需人工复核。
- 支持人工确认模式和全自动模式。
- 支持段落级重写与扩写，并保留上下文一致性检查与差异对比。
- 提供基础结构化记忆能力，包括全书摘要、角色卡、世界观设定、已发生事件摘要、章节状态。
- 提供后台管理能力，包括模板审核与发布、会员套餐配置、免费章节额度、月度额度、订单管理、手动调额、基础运营报表、敏感内容策略配置、日志留存。
- 提供内容安全、敏感题材过滤、版权声明与数据留存要求的基础支持。

## 非目标范围
- 多用户协作创作与实时协同编辑。
- 人工在线逐章精修工作台等复杂人工编辑能力。
- 高级提示词运营后台与细粒度成本统计。
- 复杂队列调度和高并发批处理优化。
- AI 配图生成与图文联动创作。
- 社区协作、评论、点赞、共创生态。
- 高级数据看板和复杂 BI 分析。
- 多格式导出能力。
- 企业级权限系统、复杂支付结算与发票体系。

## 关键用户流程
1. 用户创建小说项目，选择题材、篇幅与模板，并补充总纲、角色设定、世界观信息。
2. 用户发起续写任务，选择人工确认模式或全自动模式，并指定续写章节数，单次最多 10 章。
3. 系统按章节串行执行任务，先由故事编辑 Agent 生成 3 个剧情走向方案。
4. 资深主编 Agent 对 3 个方案进行评分，自动选择最高分方案。
5. 写手 Agent 基于选中方案和结构化记忆生成章节正文。
6. 资深读者 Agent 对正文评分并给出问题说明。
7. 若正文评分大于等于 8.0，则该版本通过；若低于 8.0，则进入回流，由写手 Agent 重写。
8. 单章最多回流 5 次；若仍未达标，则选择本章最高分版本输出，并标记“建议人工复核”。
9. 在人工确认模式下，用户逐章查看结果并决定是否采纳、继续、重写或扩写。
10. 在全自动模式下，系统连续推进后续章节，直至完成本次任务或达到 10 章上限。

## 页面/交互要点
- 项目创建页：支持选择题材、篇幅、系统模板或自定义模板。
- 项目工作台：展示总纲、角色卡、世界观、章节列表与当前任务状态。
- 章节方案面板：每章展示 3 个走向方案、主编评分明细、最终入选方案。
- 章节生成结果页：展示正文内容、读者评分、问题说明、回流次数、最终保留版本。
- 自动续写控制区：支持设置本次续写章数，最大 10 章，并显示当前模式为人工确认或全自动。
- 差异对比交互：段落级重写或扩写时，需展示修改前后差异，并提醒上下文一致性风险。
- 人工复核提示：当章节在 5 次回流后仍未达标，界面需明确标注“建议人工复核”。
- 后台模板管理页：支持模板审核、发布、标签管理和基础使用情况查看。
- 后台会员管理页：支持免费章节额度、月度额度、套餐配置、订单管理和手动调额。
- 内容安全与日志页：支持敏感内容策略配置、版权声明配置和任务日志留存查看。

## Acceptance Criteria
- 用户可以创建小说项目，并配置题材、篇幅、模板、总纲、角色设定、世界观信息。
- 用户可以发起单次续写任务，且系统限制单次最多生成 10 章。
- 每一章必须生成且仅展示 3 个剧情走向方案供后续评分。
- 系统必须使用资深主编 Agent 对 3 个剧情走向方案进行评分，并选择最高分方案进入正文生成。
- 系统必须使用写手 Agent 基于入选方案生成章节正文。
- 系统必须使用资深读者 Agent 对正文进行评分，并返回问题说明。
- 当正文评分低于 8.0 时，系统必须触发回流重写，且单章回流次数上限为 5 次。
- 单章处理完成后，系统必须保留该章所有版本中评分最高的正文作为最终版本。
- 若单章 5 次回流后仍未达到 8.0，系统必须输出最高分版本，并标记为“建议人工复核”。
- 用户可以在人工确认模式下逐章决定是否采纳生成结果。
- 用户可以在全自动模式下连续生成多章内容，直到任务完成或达到 10 章上限。
- 用户可以对已生成正文的任意段落发起重写或扩写，并查看差异对比结果。
- 系统必须具备基础结构化记忆能力，用于维持章节续写过程中的角色、世界观、事件和摘要一致性。
- 后台必须支持模板审核、会员套餐配置、免费章节额度与月度额度管理、订单管理、手动调额。
- 系统必须支持敏感内容过滤、版权声明展示和任务日志留存。
