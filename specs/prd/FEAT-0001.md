# PRD - FEAT-0001

## 背景
“智能小说协同生成平台”旨在为小说创作者提供一个由多智能体协作、并保留人工决策权的创作系统。产品核心要解决长篇或连续创作中的情节断层、设定漂移、章节质量不稳定等问题，同时支持模板化创作和会员化运营。

结合已有需求澄清，首期产品以 Web 平台为主，面向单用户创作场景，采用“小说项目 > 总纲 > 章节大纲 > 章节正文”的基本结构。系统既要支持用户逐章把控，也要支持在明确规则下的全自动续写流程。

## 最终选定方案
最终采用 `solution_1：审稿驱动型章节工坊` 作为产品基础方案。

该方案以章节为核心交付单元，围绕“生成候选方案 -> 选择最优走向 -> 生成正文 -> 评分回流 -> 保留最高分版本”构建主流程，优先满足首期快速落地、质量可控、流程清晰、便于验收的目标。

根据人类最终决策，首期产品不仅保留逐章审稿式创作体验，还必须加入受控的全自动流程：系统单次最多连续生成 10 章，每章先产出 3 个剧情走向，由“资深主编 Agent”评分选优，再进入正文生成与“资深读者 Agent”评分回流，最多回流 5 次，并保留该章评分最高版本。

## 方案修改点
相对原始 `solution_1`，最终方案必须增加和明确以下修改：

- 在人工确认模式之外，新增全自动续写模式。
- 全自动模式单次任务最多支持续写 10 章。
- 每章必须先由故事编辑 Agent 生成 3 个剧情走向方案。
- 新增“资深主编 Agent”，负责对 3 个走向方案按情节推动、故事一致性、吸引力进行评分，并自动选择最高分方案。
- 写手 Agent 基于被选中的走向方案生成章节正文。
- 资深读者 Agent 对章节正文按可读性、情绪张力、连贯性评分，并给出问题说明。
- 当正文评分低于 8.0 时，必须自动回流给写手 Agent 重写，单章最多回流 5 次。
- 若回流 5 次后仍未达标，系统仍需输出该章评分最高版本，并标记“建议人工复核”。
- 首期必须保留结构化记忆能力，至少覆盖全书摘要、角色卡、世界观设定、已发生事件摘要、章节状态。
- 首期必须具备基础任务状态展示能力，便于查看章节进度、评分、回流次数和最终结果。

## 目标用户
- 个人网文作者、轻小说作者，重视创作效率但不希望失去情节控制权。
- 需要连续产出章节草稿的内容工作者，希望借助 AI 完成方案生成、正文扩写和质量筛选。
- 平台运营人员，需要配置模板、会员权益、免费额度、订单和内容安全策略。
- 偏审稿式创作的用户，希望在关键节点人工拍板，必要时切换到全自动续写模式。

## 核心业务目标
- 提供可持续的章节级小说创作工作流，而非一次性文本生成工具。
- 通过多 Agent 分工与双重评分机制，提高章节质量和情节连贯性。
- 支持人工确认与全自动生成两种模式，兼顾可控性与产能。
- 通过模板体系、会员额度和后台运营能力支撑产品商业化。
- 为后续扩展配图、社区、数据分析和导出能力预留空间，但不纳入首期交付。

## MVP 功能范围
- 小说项目创建：支持选择题材、篇幅、模板。
- 模板能力：支持系统模板与用户自定义模板的创建、编辑、使用。
- 创作基础结构：支持总纲、章节大纲、章节正文的组织与管理。
- 大纲编辑器：支持查看、编辑章节大纲，并与故事编辑 Agent 进行近实时对话优化。
- 章节走向生成：每章生成 3 个剧情走向候选，包含核心冲突、关键事件、结尾钩子。
- 主编评分选优：由资深主编 Agent 对候选走向评分并自动选出最高分方案。
- 正文生成：写手 Agent 基于选中走向生成章节正文，并支持风格化润色。
- 读者评分回流：资深读者 Agent 对正文评分、给出问题说明，并在低于阈值时触发最多 5 次回流。
- 最高分版本保留：每章最终输出评分最高版本，未达阈值时标记人工复核建议。
- 创作模式：支持人工确认模式和全自动续写模式。
- 自动续写能力：单次最多连续生成 10 章。
- 段落级编辑：支持对任意段落发起重写、扩写，并保留上下文一致性检查与差异对比。
- 连贯性记忆：支持全书摘要、角色卡、世界观设定、已发生事件、章节状态等结构化记忆。
- 后台运营能力：支持模板审核与发布、会员套餐配置、免费章节额度/月度额度管理、订单管理、手动调额、基础运营报表、敏感内容策略配置、日志与数据留存。
- 内容安全与合规：支持敏感题材过滤、版权声明、数据留存要求。

## 非目标范围
- 多用户实时协同创作与复杂团队权限体系。
- 社区发布、评论、点赞、协作共创等社交能力。
- AI 配图生成与图文联动创作。
- 多格式导出，如 PDF、EPUB、DOCX、出版社格式包。
- 复杂支付结算、发票、渠道分销和营销活动系统。
- 高级 BI 看板与细粒度成本统计。
- 复杂队列调度和大规模并发生产优化。
- 完整移动端 App 与离线创作体验。

## 关键用户流程
1. 用户创建小说项目，选择题材、篇幅和模板，并补充世界观、角色、总纲等基础设定。
2. 用户进入章节创作流程，可先在大纲编辑器中查看或修改章节目标，并与故事编辑 Agent 讨论优化。
3. 对于单章或自动续写任务，故事编辑 Agent 为当前章节生成 3 个剧情走向方案。
4. 资深主编 Agent 对 3 个方案评分，并自动选择最高分方案进入正文生成。
5. 写手 Agent 基于选中方案与结构化记忆生成章节正文。
6. 资深读者 Agent 对正文进行评分并输出问题说明。
7. 若评分 >= 8.0，章节通过；若评分 < 8.0，则回流给写手 Agent 重写，最多 5 次。
8. 系统保留本章评分最高的正文版本；若 5 次后仍未达标，则输出最高分版本并标记建议人工复核。
9. 人工确认模式下，用户可逐章审阅、确认、重写或扩写段落后再入库。
10. 全自动模式下，系统一次最多连续推进 10 章，并在任务面板中展示每章状态、评分、回流情况与最终结果。

## 页面/交互要点
- 项目创建页：支持题材、篇幅、模板选择，并清晰展示模板来源与核心约束。
- 小说工作台：以章节为中心组织创作流程，展示总纲、章节列表、当前状态和记忆摘要。
- 大纲编辑器：支持章节目标编辑、与故事编辑 Agent 的近实时对话、候选方案查看与确认。
- 候选方案面板：固定展示每章 3 个走向方案，以及资深主编 Agent 的评分结果与选中原因。
- 正文审稿区：展示写手 Agent 生成的正文、资深读者 Agent 的评分和问题说明。
- 回流状态展示：明确展示当前 revision 次数、历史最高分、通过阈值、是否建议人工复核。
- 段落级操作：用户可选中段落发起重写/扩写，并看到差异对比与上下文一致性提示。
- 自动续写任务面板：展示最多 10 章的批次任务状态、当前处理章节、失败章节、最高分版本结果。
- 记忆卡片区：展示并维护全书摘要、角色卡、世界观设定、事件摘要、章节状态，供用户理解系统上下文依据。
- 后台管理页：支持模板审核、会员套餐与额度配置、订单管理、手动调额、内容安全策略和日志留存查看。

## Acceptance Criteria
- 系统可创建小说项目，并支持选择题材、篇幅、系统模板或用户自定义模板。
- 系统可维护总纲、章节大纲、章节正文三层创作结构。
- 每章生成流程中，故事编辑 Agent 必须稳定输出 3 个剧情走向候选。
- 资深主编 Agent 必须对 3 个候选走向给出评分和说明，并自动选出最高分方案。
- 写手 Agent 必须基于选中方案生成章节正文。
- 资深读者 Agent 必须对章节正文输出可读性、情绪张力、连贯性评分及问题说明。
- 当章节评分低于 8.0 时，系统必须自动触发回流重写，且单章回流次数不超过 5 次。
- 每章最终必须保留评分最高版本；若 5 次回流后仍未达标，系统必须输出最高分版本并标记“建议人工复核”。
- 系统必须支持人工确认模式和全自动续写模式两种创作模式。
- 全自动续写模式下，单次任务最多可连续生成 10 章，并可在任务面板查看各章状态。
- 系统必须支持段落级重写和扩写，并提供差异对比与上下文一致性检查结果。
- 系统必须具备结构化记忆能力，至少包含全书摘要、角色卡、世界观设定、已发生事件摘要、章节状态。
- 后台必须支持模板审核、会员套餐配置、免费章节额度/月度额度调整、订单管理、手动调额、基础运营报表。
- 系统必须提供敏感题材过滤、版权声明与日志/数据留存能力。
