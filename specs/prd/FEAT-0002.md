# PRD - FEAT-0002

## 背景
当前后台“套餐与额度”页面存在两类问题：
- 展示语义混乱：额度概览展示的是 `active_plan_id` 与用户剩余额度，但运营需要看到的是默认套餐的名称与规则。
- 权益模型不清晰：套餐规则仍使用旧的“免费章节 / 月度额度”命名，无法准确表达“每日可用额度、每月可用额度、后台额外赠送额度”三类不同来源。

本次功能要对套餐规则、账户额度和后台界面进行一次一致性重设计，确保运营能准确维护套餐并理解额度扣减顺序。

## 产品目标
- 将套餐权益统一定义为“日免费章节 + 月免费章节”。
- 将账户额外赠送额度独立定义为“免费章节”。
- 让后台“额度概览”稳定展示默认套餐信息，并与套餐列表保持一致。
- 让套餐维护改为“列表查看 + 弹框编辑”的工作流。

## 核心概念

### 1. 默认套餐
- 系统级默认生效套餐。
- 用于新用户初始化套餐规则。
- 在后台“额度概览”中直接展示其名称与规则。

### 2. 套餐规则
- 套餐名称
- 日免费章节
- 月免费章节
- 套餐描述

套餐规则是模板，不等于用户账户当前剩余额度。

### 3. 用户额度账户
每个用户账户持有三类可消费额度：
- `daily_remaining`：当日剩余章节数
- `monthly_remaining`：当月剩余章节数
- `bonus_remaining`：赠送免费章节剩余数

### 4. 扣减顺序
单次章节消耗时必须按以下顺序扣减：
1. 优先扣减当日剩余
2. 当日不足时扣减当月剩余
3. 当日和当月都不足时扣减赠送免费章节

## 用户故事
- 作为运营，我希望在额度概览中直接看到默认套餐名称、日免费章节、月免费章节，这样我能立即确认平台默认权益。
- 作为运营，我希望在套餐列表中点击“编辑套餐”后用弹框修改套餐，而不是在页面底部混合编辑，这样我能更清楚地维护每个套餐。
- 作为管理员，我希望给目标账号补充“赠送免费章节”，并让它在日/月额度耗尽后再被消耗，这样我能做补偿而不破坏套餐规则。

## 功能范围

### 后台额度概览
展示以下信息：
- 默认套餐名称
- 默认套餐的日免费章节
- 默认套餐的月免费章节
- 当前目标账号的当日剩余
- 当前目标账号的当月剩余
- 当前目标账号的赠送免费章节剩余
- 套餐总数

说明：
- 若未选择目标账号，则展示全局默认账户视角的额度账户。
- 默认套餐信息必须来自默认套餐实体本身，而不是从用户剩余额度推导。

### 套餐列表
每个套餐卡片展示：
- 套餐名称
- 套餐 ID
- 日免费章节
- 月免费章节
- 描述
- “编辑套餐”按钮
- “设为默认套餐”按钮（当前默认套餐显示“当前默认”徽标）

### 套餐编辑弹框
弹框标题：
- 编辑时：`编辑套餐`
- 新建时：`新建套餐`

字段：
- 套餐名称
- 日免费章节
- 月免费章节
- 描述

行为：
- 点击套餐列表中的“编辑套餐”打开弹框并回填当前套餐数据。
- 点击“新建套餐”打开空白弹框。
- 点击“保存套餐”提交接口并关闭弹框。
- 保存成功后刷新套餐列表与额度概览。

### 额度调整
额度调整区改为支持：
- 目标账号
- 赠送免费章节调整值

可选扩展：
- 若保留对当日/当月剩余额度的直接调整，应明确标注为“手动修正剩余额度”，避免与赠送额度混淆。

本次建议优先保留三项调整能力：
- 当日剩余调整
- 当月剩余调整
- 赠送免费章节调整

但 UI 文案必须明确区分“剩余额度修正”和“赠送额度发放”。

## 页面交互要求
- “套餐编辑台”不再常驻页面，而以模态弹框出现。
- 套餐列表中的主按钮文案由“保存套餐”改为“编辑套餐”。
- 额度概览中的文案统一改为：
  - 默认套餐
  - 日免费章节
  - 月免费章节
  - 赠送免费章节
  - 当日剩余
  - 当月剩余
- 页面中不再出现“免费章节 / 月度额度”这组旧术语。

## 验收标准
- 额度概览中的默认套餐名称、日免费章节、月免费章节，与套餐列表中对应默认套餐完全一致。
- 套餐列表中点击“编辑套餐”后，会打开弹框，而不是直接在页内编辑。
- 编辑套餐保存成功后，套餐列表和额度概览同步刷新。
- 用户额度账户中新增赠送免费章节字段，并可被后台查看和调整。
- 章节额度扣减顺序符合“当日 -> 当月 -> 赠送免费章节”。
- 当目标账号当日和当月剩余额度均为 0、但赠送免费章节大于 0 时，仍允许继续生成章节。
- 当三类额度都为 0 时，创建任务或执行任务应返回 `QUOTA_EXCEEDED`。

## 风险与兼容
- 旧数据中的 `free_remaining` 需要迁移为新语义；建议映射为 `bonus_remaining`，避免误把历史赠送值当作新的日额度。
- 旧的 `monthly_remaining` 可迁移为新的 `monthly_remaining`，但若历史数据本质上是“总月额度”，需补充重置策略。
- 前端所有“免费章节/免费额度/月度额度”文案需要统一替换，避免运营误读。
