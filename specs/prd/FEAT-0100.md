# PRD - FEAT-0100

## 背景
“智能小说协同生成平台”面向持续创作型小说作者，核心问题不是单次生成效率，而是长篇创作中的情节连贯性、设定一致性和创作可控性。原始需求明确提出以多种 AI Agent 协作完成“规划、写作、评审、回流修订”的闭环，同时保留用户作为最终决策者的控制权。

首期产品不将多 Agent 直接暴露为复杂编排平台，而是收敛为可配置的阶段式创作流程。系统在前台提供项目创建、大纲编辑、章节生成与局部改写能力，在后台提供模板和会员权益配置能力，以最小可用闭环验证创作价值与商业化基础。

## 目标用户
- 持续进行网文或小说创作的个人作者。
- 希望借助 AI 提升创作效率，但仍需掌控情节方向、风格和章节定稿权的创作者。
- 需要通过模板和会员权益进行运营管理的平台管理员。

## 业务目标
- 帮助用户更高效地产出结构化、连贯的小说内容。
- 通过“故事编辑 Agent + 写手 Agent + 资深读者 Agent”的阶段式协作降低内容失控风险。
- 通过结构化记忆机制减少角色设定遗忘、世界观漂移和剧情断层。
- 通过默认人工确认机制确保用户始终保有关键创作决策权。
- 通过模板管理和会员权益配置支撑平台基础运营与配额管理。

## 用户故事
- 作为创作者，我希望在创建项目时选择题材、篇幅和模板，以便快速进入适合的创作起点。
- 作为创作者，我希望在大纲编辑器中查看和编辑章节大纲，并与故事编辑 Agent 讨论多个走向方案，以便先确定方向再生成正文。
- 作为创作者，我希望在每章生成前先选择或修改 AI 给出的概要方案，以便保持作品在我的控制之下。
- 作为创作者，我希望写手 Agent 基于已确认大纲和既有设定生成正文，以便减少重复劳动并提高成稿速度。
- 作为创作者，我希望系统在生成后自动进行质量评分并在低分时回流修订，以便提升章节质量并减少明显问题。
- 作为创作者，我希望对已生成章节执行确认、整章重写、段落扩写和段落重写，以便针对局部内容继续迭代。
- 作为管理员，我希望在后台维护模板和会员权益配置，以便管理免费次数、月度额度等基础运营规则。

## MVP 功能范围
- 小说项目创建：支持题材、篇幅、模板选择。
- 项目级大纲编辑器：支持章节大纲查看、编辑，以及与故事编辑 Agent 的对话式迭代。
- 章节方案生成：故事编辑 Agent 为目标章节输出多个走向和概要方案，用户选择或修改后定稿。
- 章节正文生成：写手 Agent 基于已确认大纲、结构化记忆和已生成章节摘要生成正文。
- 章节质量评审：资深读者 Agent 按预设维度评分，并在低于阈值时触发有限次数自动回流。
- 人工审批与局部改写：用户可确认章节、要求整章重写、对段落执行扩写或重写。
- 结构化记忆管理：至少覆盖角色信息、世界观设定、关键事件、章节摘要，并供生成与评审流程复用。
- 自动生成模式：作为可选项提供连续执行能力，但用户必须可以随时暂停和接管。
- 管理后台最小能力：模板管理、会员档位配置、免费次数和月额度配置。

## 非目标范围
- AI 配图生成。
- 社区协作、评论互动和作品分享。
- 多人实时协作编辑和团队权限体系。
- 数据看板、复杂运营分析和 BI 能力。
- PDF、Word、电子书等多格式导出链路。
- 支付、结算、发票及财务系统集成。
- 可视化工作流编排平台。
- 多语言创作市场支持。

## 验收标准（Acceptance Criteria）
- 用户可以成功创建小说项目，并在创建时完成题材、篇幅和模板选择。
- 用户可以在项目内查看章节列表并编辑章节大纲。
- 用户可以在大纲编辑过程中与故事编辑 Agent 进行多轮对话，并获得多个章节走向/概要候选方案。
- 系统要求用户先对章节方案执行选择或修改定稿，再进入正文生成。
- 写手 Agent 生成章节正文时必须使用已确认大纲和项目结构化记忆。
- 资深读者 Agent 必须对生成章节输出评分结果，并在低于阈值时触发有限次数自动回流，而非无限循环。
- 当自动回流达到上限后，系统必须停止自动重试并将决定权交还给用户。
- 用户可以对章节执行“确认”“整章重写”“段落扩写”“段落重写”操作。
- 自动生成模式下，系统可以连续推进章节流程，但用户可随时暂停并人工接管。
- 管理员可以在后台新增、编辑、启停项目初始化模板。
- 管理员可以配置会员档位中的免费次数、月额度等基础权益，且权益规则不硬编码在前台流程中。

## 页面/交互要点
- 项目创建页：突出题材、篇幅、模板三项核心选择，模板需区分系统模板与用户自定义模板入口。
- 项目工作台：以“项目信息 + 章节列表 + 当前流程状态”组织主界面，帮助用户明确当前章节所处阶段。
- 大纲编辑器：支持按章节查看和编辑大纲，右侧或侧边栏提供与故事编辑 Agent 的对话区，便于边讨论边修订。
- 章节方案选择区：对同一章节展示多个候选走向/概要，用户可以直接选择、局部修改后确认。
- 正文生成页：需清晰显示本次生成依赖的已确认大纲与关键信息摘要，增强用户可控感。
- 评审结果区：展示资深读者 Agent 的评分和问题摘要，并明确当前是否触发回流、剩余回流次数和下一步动作。
- 章节阅读与编辑区：段落级操作需就近触发，支持对任意段落发起扩写或重写，不打断整章阅读体验。
- 自动生成控制：需要在明显位置提供启动、暂停、接管状态与额度消耗提示，防止用户失去流程控制。
- 后台管理页：模板管理与会员权益管理应拆分为独立模块，强调配置清晰、状态明确和最小操作路径。
