# Requirement Card - FEAT-0100

## 原始需求
我计划开发一个名为“智能小说协同生成平台”的系统。其核心目标是构建一个由“多种AI智能体协作 + 人工审核决策”驱动的创作环境，用于生成情节连贯、可深度定制的小说。系统前端需要支持用户创建项目时选择小说类型（如言情、玄幻、架空、恐怖）、篇幅（长篇/短篇）以及模板（系统提供或用户自定义）。系统还需要一个后台，用于管理模板和会员制度，例如免费次数、月度额度等权益配置。

创作流程的重点是解决长篇创作中的情节连贯性与记忆断层问题。设想中的流程为：先由“故事编辑 Agent”生成多个后续章节走向和主题概要供用户选择；方向确定后，再由“写手 Agent”扩写、补细节并完成风格化润色；最后由“资深读者 Agent”对章节进行质量评估与打分，若低于标准则自动回流至前置环节重写或补充，形成质量闭环。

系统要求用户始终拥有最高决策权。默认模式下，每个章节内容都需要人工确认；但用户也可以选择“自动生成”模式，让 AI 连续执行。系统还需提供大纲编辑器，支持用户查看和编辑任意章节大纲，并在编辑过程中与“故事编辑 Agent”实时对话、反复优化方案，最终由用户定案。对于已生成内容，用户还可以对任意段落发起“重写”或“扩写”。

此外，系统需要考虑后续扩展能力，例如配图生成、社区协作、数据看板、多格式导出等。在技术上，重点是长上下文记忆管理与后台模板/会员运营能力。

## 当前理解
这是一个面向小说创作者的 AI 协同写作平台。系统不是单次文本生成工具，而是围绕“项目 - 大纲 - 章节 - 段落”构建持续迭代式创作流程。产品核心价值不只是生成速度，而是通过多智能体分工、状态记忆和人工决策节点，提升小说创作的连贯性、可控性和产出质量。

从业务结构看，至少包含两大域：
1. 创作域：项目创建、模板选择、大纲规划、多 Agent 协作生成、章节审核、段落重写/扩写、自动回流修订。
2. 运营域：模板管理、会员权益管理、配额控制，以及后续可能扩展的运营数据能力。

从产品机制看，用户默认是“最终审批者”，AI 是辅助执行者。自动生成模式属于增强选项，而不是替代用户控制权的默认模式。

## 目标用户
- 有持续创作需求的网文作者、小说作者、内容工作室成员。
- 希望通过 AI 提升创作效率，但仍需保留选题、风格、情节方向控制权的用户。
- 需要模板化生产、会员权限控制与项目管理能力的平台运营方或内容团队。

## 核心业务目标
- 帮助用户更高效地产生结构化、连贯的小说内容。
- 通过多 Agent 协作分工，降低单模型在情节规划、细节扩写和质量把关上的失真风险。
- 通过长上下文记忆或替代记忆机制，减少长篇写作中的人物设定遗忘、世界观漂移和剧情断层。
- 通过人工确认节点，确保用户始终保有关键创作决策权。
- 通过模板和会员后台，支持平台商业化运营与权益管理。

## 不明确问题
- 平台首期是面向 C 端作者，还是面向工作室/团队协作场景，两者权限模型差异较大。
- “实时对话讨论”的范围不清晰，是仅针对大纲编辑器内单章节讨论，还是可跨章节引用上下文进行持续对话。
- 多 Agent 是真实独立编排节点，还是产品层抽象的多阶段生成流程；这会直接影响架构复杂度。
- “资深读者 Agent”的评分维度和阈值未定义，例如连贯性、文风一致性、节奏、人物行为合理性分别如何计分。
- 自动回流机制未明确边界，需确认最大回流次数、失败兜底策略以及是否允许用户介入中断。
- 模板的定义粒度不清晰，可能包括题材模板、结构模板、文风模板、章节模板，后台设计会因此不同。
- 会员制度仅提到免费次数和月度额度，尚不清楚是否还包含模型等级、并发上限、章节长度上限等权益。
- 自定义模板是否允许用户私有保存、复制官方模板、共享给他人，目前没有说明。
- “长篇/短篇”只是创建时的标签，还是会影响后续生成策略、上下文管理和收费规则，尚未明确。
- 扩展能力中提到配图、社区、数据看板和导出，但未说明哪些需要纳入首期架构预留。

## 暂定假设
- 首期以单用户创作项目为主，不处理多人实时协作编辑。
- 首期聚焦中文小说创作场景，不同时支持多语言市场。
- 创作主链路按“项目创建 -> 大纲讨论/定稿 -> 章节生成 -> 章节评分 -> 人工确认/回流修订”展开。
- 故事编辑 Agent、写手 Agent、资深读者 Agent 在首期可实现为可配置的多阶段 AI 工作流，不要求用户感知底层技术编排细节。
- 默认模式下，章节级内容必须经用户确认后才能进入下一章节。
- 自动生成模式仍保留用户随时暂停、修改大纲和接管流程的能力。
- 段落级“重写/扩写”先只作用于已生成正文，不扩展到大纲或角色设定文档。
- 模板后台和会员后台为平台管理员使用，不在首期开放复杂的多角色后台权限体系。

## 建议 MVP
- 支持用户创建小说项目，并选择题材、篇幅和模板。
- 提供项目级大纲编辑器，允许用户查看、编辑章节大纲，并与故事编辑 Agent 进行对话式迭代。
- 支持故事编辑 Agent 生成多个章节走向/概要方案供用户选择。
- 支持写手 Agent 基于已定大纲生成章节正文。
- 支持资深读者 Agent 对章节进行评分，并在低于阈值时触发有限次数的自动回流。
- 支持用户对章节结果执行人工确认、拒绝重写、段落扩写和段落重写。
- 提供基础的长上下文记忆方案，例如项目设定、角色卡、已发生关键事件摘要、章节摘要等结构化记忆。
- 提供管理员后台的最小可用能力：模板管理、会员档位配置、免费次数/月额度配置。

## Out of scope
- AI 配图生成。
- 社区协作、作品分享、评论互动。
- 数据看板与复杂运营分析。
- 多格式导出到电子书、PDF、Word 等完整发布链路。
- 多人实时协作编辑、团队权限管理。
- 复杂结算体系、支付接入、发票或财务系统集成。
- 细粒度工作流编排可视化配置平台。

## 待确认项
- 首期目标用户和商业模式优先级：个人作者、工作室，还是平台运营方。
- 首期必须交付的前台功能与后台功能边界。
- 模板的准确分类、字段结构和生命周期。
- 会员权益模型的完整定义与计量单位。
- 章节评分标准、阈值、回流次数和人工介入规则。
- 自动生成模式的终止条件、风险提示和消耗规则。
- 长上下文记忆方案的产品可见形态，例如是否允许用户查看/编辑角色卡、世界观卡、事件时间线。
- 是否需要在首期支持自定义模板创建与复用。
