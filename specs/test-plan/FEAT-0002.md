# Test Plan - FEAT-0002

## 测试目标
验证套餐规则、默认套餐展示、用户额度账户和赠送免费章节逻辑在后台和扣减流程中保持一致。

## 测试范围
- 后端数据模型与接口返回
- 默认套餐展示一致性
- 套餐编辑弹框交互
- 额度调整与扣减顺序
- 兼容旧数据迁移后的读写行为

## 核心用例

### 1. 默认套餐概览一致性
- 前置条件：存在多个套餐，默认套餐为 `plan-basic`
- 操作：打开后台“套餐与额度”页
- 预期：
  - 额度概览展示的默认套餐名称、日免费章节、月免费章节，与套餐列表中 `plan-basic` 卡片一致
  - 不再展示旧字段名“免费章节 / 月度额度”

### 2. 编辑套餐弹框
- 前置条件：套餐列表存在至少 1 个套餐
- 操作：点击“编辑套餐”
- 预期：
  - 打开弹框
  - 表单回填当前套餐字段
  - 保存成功后弹框关闭
  - 列表和额度概览刷新为最新值

### 3. 新建套餐弹框
- 操作：点击“新建套餐”
- 预期：
  - 打开空白弹框
  - 可输入套餐名称、日免费章节、月免费章节、描述
  - 保存成功后列表新增套餐

### 4. 设置默认套餐
- 操作：在套餐列表中将非默认套餐设为默认
- 预期：
  - 默认套餐徽标切换
  - 额度概览同步显示新的默认套餐名称和规则

### 5. 赠送免费章节展示与调整
- 前置条件：目标账号 `bonus_remaining = 0`
- 操作：后台对目标账号增加 `bonus_delta = 5`
- 预期：
  - 额度概览显示赠送免费章节为 5
  - 不影响默认套餐规则值

### 6. 扣减顺序
- 前置条件：
  - `daily_remaining = 1`
  - `monthly_remaining = 2`
  - `bonus_remaining = 3`
- 操作：连续消费 4 章
- 预期：
  - 第 1 章扣日额度
  - 第 2-3 章扣月额度
  - 第 4 章扣赠送免费章节

### 7. 日重置
- 前置条件：
  - 用户绑定套餐日免费章节为 5
  - 当前 `daily_remaining = 0`
  - `last_daily_reset_at` 为昨天
- 操作：读取额度或发起任务
- 预期：
  - 自动重置 `daily_remaining = 5`

### 8. 月重置
- 前置条件：
  - 用户绑定套餐月免费章节为 50
  - 当前 `monthly_remaining = 0`
  - `last_monthly_reset_at` 为上月
- 操作：读取额度或发起任务
- 预期：
  - 自动重置 `monthly_remaining = 50`

### 9. 额度不足拦截
- 前置条件：
  - `daily_remaining = 0`
  - `monthly_remaining = 0`
  - `bonus_remaining = 0`
- 操作：创建续写任务
- 预期：
  - 返回 `QUOTA_EXCEEDED`

## 回归重点
- 订单与套餐列表接口仍能正常返回
- 已有管理员可继续查看用户列表并选择目标账号
- 创作者工作台读取额度接口不会因字段重命名而崩溃

## 建议验证方式
- 后端：pytest 单测覆盖服务层扣减、重置、迁移逻辑
- 前端：组件级测试或人工验证后台“套餐与额度”页面
- 集成：管理员修改套餐后刷新页面检查概览与列表一致性
