# Test Plan - FEAT-0100

## 测试目标
- 验证“智能小说协同生成平台”MVP 主链路满足需求：项目创建、大纲协作、章节方案确认、正文生成、质量评审、有限回流、人工确认与局部改写。
- 验证用户始终保有关键决策权，默认流程不会绕过人工确认节点。
- 验证结构化记忆、会员额度、模板配置和自动生成控制在关键流程中被正确使用。
- 验证后台模板与会员权益配置可支撑前台创作流程，且规则来源于配置而非流程硬编码。

## 范围
- 前台创作域
  - 项目创建：题材、篇幅、模板选择。
  - 项目工作台与章节状态流转。
  - 大纲编辑器：章节大纲查看、编辑，与故事编辑 Agent 对话式迭代。
  - 章节方案生成与用户确认。
  - 章节正文生成。
  - 资深读者 Agent 评分与低分自动回流。
  - 用户确认章节、整章重写、段落重写、段落扩写。
  - 自动生成模式的启动、暂停、接管和阻塞状态展示。
- 后台管理域
  - 模板新增、编辑、启停。
  - 会员档位与免费次数、月额度配置。
- 非功能与约束验证
  - 回流次数上限生效，避免无限循环。
  - 生成类接口额度校验生效。
  - 结构化记忆在生成、评审、重写后可复用与更新。
  - 审计与错误返回满足基本可追溯性。

## 风险点
- 情节方案未确认即进入正文生成，破坏“用户最高决策权”要求。
- 结构化记忆未参与生成或确认后未更新，导致长篇一致性失效。
- 评审低分回流未正确限制次数，可能出现无限自动重试。
- 自动生成模式无法暂停或接管，导致流程失控与额度风险。
- 段落重写/扩写覆盖原版本且未保留历史，造成内容丢失。
- 会员权益、免费次数或月额度校验缺失，导致超额调用。
- 后台模板/会员配置变更无法影响前台流程，说明存在硬编码风险。
- 系统模板与用户模板读取模型不一致，导致项目初始化失败或结果偏差。

## 测试策略
- 基于需求验收标准设计主链路测试，优先覆盖“创建项目 -> 大纲讨论与定稿 -> 生成正文 -> 评审回流 -> 用户确认/改写 -> 记忆更新”闭环。
- 对关键状态机和业务规则采用数据驱动测试，覆盖正常路径、边界条件和失败分支。
- 对 AI Agent 相关能力在单元/集成层使用可控桩或契约响应，验证输入上下文、状态流转和回流决策，不将模型内容质量本身作为稳定自动化断言对象。
- 对结构化记忆、额度扣减、回流计数、章节版本保留等持久化行为进行集成验证。
- 对前后台关键用户流程设计 e2e 场景，验证页面或 API 串联后的真实业务结果。
- 对错误处理重点验证：未确认大纲禁止生成、配额不足禁止生成、回流次数耗尽后阻塞并交还用户决策、停用模板不可继续被新项目使用。

## 测试层次（unit/integration/e2e）
### unit
- 项目创建参数校验：题材、篇幅、模板状态、自动生成开关。
- 章节状态机规则：未确认大纲禁止生成、低分触发回流、达到上限转为阻塞状态。
- 评审回流决策逻辑：阈值比较、剩余重试次数、失败兜底。
- 会员额度规则：免费次数、月额度扣减与不足判定。
- 模板启停与模板读取统一模型。
- 结构化记忆组装与确认后更新逻辑。
- 章节版本生成规则：整章重写、段落重写、段落扩写生成新版本而非覆盖。

### integration
- `POST /api/v1/projects` 到项目、模板、会员额度校验的联动。
- 大纲编辑、故事方案生成、方案确认、正文生成、评审、回流、确认的完整接口联调。
- 自动生成启动/暂停/接管与工作流状态查询联动。
- 段落重写/扩写与章节版本、MemoryRecord 更新联动。
- 后台模板和会员配置变更后，前台查询与生成流程正确反映新配置。
- 审计日志和额度台账在关键动作后正确记录。

### e2e
- 创作者手动模式完成单章创作闭环。
- 创作者在大纲编辑器中与故事编辑 Agent 多轮讨论并选择方案后生成正文。
- 章节评审低分后自动回流，达到上限后停止并要求用户决策。
- 用户对段落执行扩写/重写并确认新版本。
- 自动生成模式连续推进章节，用户中途暂停并接管。
- 管理员维护模板与会员权益后，创作者使用新配置完成项目初始化与生成。
